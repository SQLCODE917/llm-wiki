This chapter explains JavaScript's function naming mechanics, contrasting `const`-bound anonymous functions with named function expressions using the `function` keyword. Key topics include:

- **Function declarations** vs. expressions and hoisting behavior
- **Named function expressions** where the function's actual name differs from its binding
- **Combinators** like `compose` for function composition
- **Function decorators** like `not` that modify function behavior
- **Partial application** patterns using `mapWith`
- **Magic names** like `this` and `arguments` in function contexts

See (raw/javascriptallonge.pdf p.62-78) for detailed examples.

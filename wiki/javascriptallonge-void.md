## Source record

A chapter on JavaScript's void operator, explaining how it evaluates to undefined regardless of the input value.

## Key supported claims

- JavaScript's void operator always evaluates to undefined, regardless of input (raw/javascriptallonge.pdf p.35-35).
- Using void 0 is the conventional and reliable way to generate undefined values in JavaScript (raw/javascriptallonge.pdf p.35-35).
- Undefined values can be generated by evaluating functions that don't return a value or by writing undefined directly, but void 0 is preferred for consistency (raw/javascriptallonge.pdf p.35-35).

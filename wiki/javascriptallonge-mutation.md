## Source record

This chapter explains how mutation works in JavaScript, covering arrays, objects, aliases, and assignment semantics, with possible nuances.

## Key supported claims

- Arrays and objects can mutate, changing structure while keeping identity (raw/javascriptallonge.pdf p.141-157).
- In JavaScript, almost every value can mutate, but identities stay the same (raw/javascriptallonge.pdf p.141-157).
- Alias bindings refer to the same value; mutation changes that shared value (raw/javascriptallonge.pdf p.141-157).

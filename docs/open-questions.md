# Open Questions Awaiting Experimentation

Decisions deliberately deferred until a greater variety of sources exists —
per-source tuning now would be premature optimization. Each entry names the
experiment that resolves it. Resolved entries move to the bottom table with
their number and outcome, so this file doubles as a decision log.

| # | Question | From | Experiment that resolves it | Status |
|---|---|---|---|---|
| 1 | **Figure-OCR thresholds.** Confidence ≥0.5 + ≥12 chars let photo noise through: on the first live extraction, 18 of 35 coffee-photo images passed (shop signage, an Italian newspaper) alongside exactly one genuine text-bearing figure (p.24, partial-functions notation). Follow-up evidence (2026-06-12): the noise *reached the wiki* — the model quoted a marker line verbatim into a chapter page ("ANTIDOTE DECAF") despite the schema forbidding it. Boundary fix shipped: `write_page` strips pipeline-marker lines from all content, so OCR plumbing can no longer enter the wiki regardless of model behavior. | PDF ingestion design | Ingest a screenshot/diagram-heavy PDF; measure precision/recall of `usable_text` filters; consider language/coherence heuristics only if the (now boundary-enforced) marker handling proves insufficient. | Open — noise path sealed at the wiki boundary |
| 2 | **Wiki churn / page granularity at book scale.** First live evidence (chunk 3): left unconstrained, the model gave every term a page (`undefined`, `nan`, ...) and exhausted its iteration budget before `finish_chunk`. Mitigated with a selectivity rule in the map prompt ("chapter page + at most 3 others") and a bigger iteration budget; whether the resulting granularity is *good* still needs the full-ingest review. | PDF ingestion design | Inspect the wiki and per-chunk transcripts after the first full `javascriptallonge.pdf` ingest; run `lint`; count rewrites per page and thin pages. | Open — live evidence 2026-06-11 |
| 3 | **Digest growth.** 20 chunks × ≤1.5K-char notes ≈ 30K chars (~7.5K tokens) approaches the integrate run's budget. First book came in at ~2K tokens with the note cap — no pressure observed. | PDF ingestion design | Check the integrate transcript for compaction events on a longer book; if phase ≥2 fires, add a two-stage reduce or lower the note cap. | Open — no pressure at 20 chunks |

| 9 | **Terminal-in-batch swallows sibling failures.** Live (chunk 5): the model batched `write_page×3 + finish_chunk` in one turn; one write failed (invalid name `IIFE`) but `finish_chunk` succeeded, so the run ended before the model saw the error — page silently missing, notes over-claim. This is documented forge behavior (ADR-005). Mitigated with a system-prompt rule (finish tools only in a turn of their own); lint's "concepts mentioned but lacking a page" review is the catch-all. | Observed live, 2026-06-11 | Watch whether the prompt rule holds across future ingests; if violations recur, propose an upstream forge option (block terminal in mixed batches) rather than wrapping the runner. | Open — mitigated |
| 10 | **Destructive rewrites during repair.** Live, twice (post-book lint): the model rewrote existing pages it had never read this run, thinning seven pages total to stub length (prompt-only rules did NOT hold — the second occurrence happened with the read-before-rewrite instruction in the prompt). Content restored from run transcripts both times (every write_page is logged with full args — observability as backup). **Now enforced structurally**: write_page raises a corrective tool error when the page exists and wasn't read this run (shared read-tracker between read_page/write_page, all five workflows; new pages unaffected; regression-tested). | Observed live, 2026-06-12 | Confirm zero content-loss across the next several lint/ingest runs; consider committing `wiki/` to git after each operation as the pattern doc recommends (defense in depth + history). | Mitigated structurally — verifying |
| 11 | **Hub synthesis misranks salience (primacy bias).** Measured, designed, and implemented same-day (docs/2026-06-12-deterministic-salience-design.md): the integrate/lint runs now receive a code-computed salience block (inbound links + recorded ingest writes; combined-score ranking after the live preview showed fresh wikis are under-linked). First live reintegrate: 14/16 hub key-entries drawn from the report (2 stray carryovers from the old hub: ecmascript-2015, closure); real center-of-mass pages (combinator, linked-lists, array-destructuring, generator) replaced front-matter picks. Residual leanpub-ranking and the stray-carryover problem were closed by #12. | Hub review, 2026-06-12 | Superseded by #12's structural fix; re-evaluate signal weighting after the next book. | Mitigated — superseded by #12 |
| 13 | **Normalized locator context.** The strict citation parser treats `normalized:L12-L20` as valid only when it appears in the same citation span as a `raw/...` source, e.g. `(raw/article.md normalized:L12-L20)`. This avoids guessing from page frontmatter, but future evidence gates may find existing or model-authored pages that use standalone normalized locators with an obvious page-level source context. | Strict citation parser implementation, 2026-06-14 | Run the citation parser over the live wiki before enabling strict evidence gates; if standalone normalized locators are common and unambiguous, design a page-context rule before `fail` mode. | Open — parser is conservative |
| 14 | **Contradiction candidate-pair quality.** The first implemented selector uses deterministic signals (shared sources, direct links, shared raw citations, keyword overlap) and caps the audited set before invoking the model. | 15 | **Uncited claim detection precision.** Grounding audit v1 deliberately selects citation-bearing claim lines and skips fatal deterministic evidence failures before model judgment.
